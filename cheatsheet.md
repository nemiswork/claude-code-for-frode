# Claude Code Cheatsheet

*Print this. Tape it to your monitor. No judgment.*

---

## Starting & Stopping

| Action | Command |
|--------|---------|
| Start Claude Code | `claude` |
| Start in a specific folder | `cd ~/my-folder && claude` |
| Exit | Type `exit` or press `Ctrl+D` |
| Cancel current operation | `Ctrl+C` |
| Clear screen | `Ctrl+L` |

## Everyday Commands

Just type these naturally — no special syntax:

```
Read [filename] and summarize it
Search the web for [topic]
Write a script that [does something]
Create a file called [name] with [content]
What files are in this directory?
Explain what this code does
Make it better / simpler / shorter
```

Read [file.csv] and analyze it

## Pro Tips

| Tip | Why |
|-----|-----|
| Use `CLAUDE.md` files | Gives Claude persistent context about your project |
| Be specific | "Analyze column B" beats "analyze the data" |
| Iterate | Say "change X" instead of re-explaining everything |
| Ask Claude to explain | "Explain this code in simple terms" |
| Ask Claude what it can do | "What features should I try next?" |

## When Something Goes Wrong

| Problem | Solution |
|---------|----------|
| Claude seems stuck | Press `Ctrl+C`, then rephrase |
| Output looks wrong | "That's not right, I wanted [X] instead" |
| Too much text | "Give me the short version" |
| Made unwanted changes | "Undo the last change" or use Git |
| Session feels confused | Type `exit`, then start fresh with `claude` |
